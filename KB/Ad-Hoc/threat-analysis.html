CIA Triad

หลักการเป้าหมายมาตรการภัยคุกคาม
Confidentialityข้อมูลเข้าถึงได้้เฉพาะผู้มีสิทธิ์Encryption, Access Control, MFAData breach, MITM
Integrityข้อมูลไม่ถูกแก้ไขโดยไม่ได้้รับอนุญาตHashing, Digital Signature, ChecksumSQL Injection, log tampering
Availabilityระบบพร้อมใช้งานเสมอRedundancy, Backup, Auto-scalingDDoS, Ransomware, HW failure
ตัวอย่างการประยุกต์ใช้: ระบบธนาคารออนไลน์ต้องใช้ TLS (Confidentiality), มี checksum ทุกรายการ (Integrity), และมี failover cluster (Availability)

YARA Rules

LockBit
PowerShell Download
rule LockBit_Ransomware {
  meta:
    description = "Detect LockBit ransomware strings"
  strings:
    $s1 = "LOCKBIT" nocase
    $s2 = "\\aacs\\" nocase
    $s3 = ".lockbit" ascii
    $s4 = "bitcoin" ascii
  condition:
    2 of ($s1,$s2,$s3) or $s4
}
rule Suspicious_Powershell_Download {
  meta:
    description = "Detect PowerShell download cradle"
  strings:
    $web = "Net.WebClient" nocase
    $down = ".DownloadString" nocase
    $invoke = "IEX" nocase
    $enc = "-enc" nocase
  condition:
    ($web and $down) and ($invoke or $enc)
}

SQL Injection → Data Exfiltration

เว็บ e-commerce มีช่องโหว่ SQL Injection ที่ search endpoint
Timeline Analysis Summary
14:23:15 ส่งคำสั่ง ' OR 1=1-- ไปที่ /api/products/search
14:23:18 Response — ข้อมูล products ทั้งหมด (ไม่ปกติ)
14:24:30 UNION SELECT username,password FROM users
14:25:00 10,000 records ถูกดึงออกใน 5 วินาที

Log Analysis

# 1. WAF alert — SQL injection pattern

  Rule: SQLI-942100 (SQL Injection - UNION)
  URI: /api/products/search?q=1'+UNION+SELECT+...

# 2. Web server log

203.0.113.5 - [24/Jun/2026:14:24:30 +0700] "GET /api/products/search?q=1'+UNION+SELECT+*+FROM+users-- HTTP/1.1" 200

# 3. SIEM — WAF + web log + DB audit
# DB audit: query จาก app user SELECT * FROM users — ผิดปกติ

ขั้นตอนวิเคราะห์

# 1. สกัด requests ของ attacker
cat nginx-access.log | grep "203.0.113.5" | awk '{print $7}' | urldecode

# 2. ทดสอบยืนยัน (ใน dev)
sqlmap -u "http://app/api/products/search?q=1" --dbs
# พบ database: ecommerce_db

# 3. ตรวจสอบ data ที่ถูก exfiltrate
cat nginx-access.log | grep "203.0.113.5" | grep "UNION"

# 4. Response size ผิดปกติ
# ปกติ ~2KB, response UNION ~500KB

Root Cause

ช่องโหว่: Raw string interpolation ใน SQL query
f"SELECT * FROM products WHERE name LIKE '%{q}%'"
ป้องกันที่ขาด: No parameterized query, DB user มีสิทธิ SELECT users
Gap: WAF ตรวจจับช้า (หลัง data ถูกขโมย) 
# ก่อน (มีช่องโหว่)
cursor.execute(f"SELECT * FROM products WHERE name LIKE '%{q}%'")


# หลัง (parameterized query)
cursor.execute("SELECT * FROM products WHERE name LIKE %s", ('%' + q + '%',))

สรุป

  1. CIA Triad — รากฐานของ security: Confidentiality, Integrity, Availability
  2. STRIDE — ใช้ DFD + STRIDE questions เพื่อ identify threat แบบ per-element
  3. PASTA — 7 ขั้นตอน เชื่อม business impact, technical analysis, risk score
  4. Analysis — Static (triage เร็ว), Dynamic (ยืนยันพฤติกรรม), Behavioral (หา unknown)
  5. Detection — Signature แม่นยำ, Anomaly จับ zero-day, Heuristic สมดุล
  6. Pyramid of Pain — ตรวจจับ TTPs (ระดับบน) ดีกว่า hash/IP (ระดับล่าง)
  7. SIEM + SOAR — รวม log ทุกแหล่ง, ใช้ rule + ML ตรวจจับ, ตอบสนองอัตโนมัติ
