CH-3: การเข้ารหัสสำหรับ Network Security

Cryptographyวิทยาการเข้ารหัสลับ — การศึกษาเทคนิคสำหรับการสื่อสารอย่างปลอดภัยในที่ที่มีผู้ไม่หวังดี, AESAdvanced Encryption Standard — มาตรฐานการเข้ารหัสแบบสมมาตรที่ใช้กันแพร่หลายที่สุด มีขนาดคีย์ 128, 192 หรือ 256 บิต, RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่, ECCElliptic Curve Cryptography — การเข้ารหัสแบบไม่สมมาตรที่ใช้โครงสร้างของElliptic Curve ให้ความปลอดภัยเทียบเท่า RSA แต่ใช้คีย์สั้นกว่า, Diffie-Hellman, Hash Functions, HMACHash-based Message Authentication Code — กลไกตรวจสอบความถูกต้องของข้อความโดยใช้ Hash Function ร่วมกับ Secret Key, Digital Signatureลายเซ็นดิจิทัล — กลไกยืนยันตัวตนผู้ส่งและความถูกต้องของข้อมูล โดยใช้การเข้ารหัสแบบไม่สมมาตร, PFSPerfect Forward Secrecy — คุณสมบัติที่ทำให้คีย์เข้ารหัสในอดีตไม่ถูกเปิดเผยแม้คีย์ส่วนตัวถูกรั่วไหล, TLSTransport Layer Security — โพรโทคอลรักษาความปลอดภัยสำหรับการสื่อสารบนอินเทอร์เน็ต สืบทอดจาก SSL 1.3 Handshake, IPsecInternet Protocol Security — ชุดโพรโทคอลสำหรับรักษาความปลอดภัยของข้อมูลในชั้น Network, SSHSecure Shell — โพรโทคอลสำหรับบริหารจัดการระบบระยะไกลอย่างปลอดภัย ใช้เข้ารหัสและตรวจสอบตัวตน, Post-Quantum Cryptographyการเข้ารหัสหลังยุคควอนตัม — อัลกอริทึมที่ต้านทานการโจมตีด้วยคอมพิวเตอร์ควอนตัม

2. การเข้ารหัสแบบ Symmetric (Symmetric Encryption)

2.1 หลักการทำงาน

ใช้ Key เดียวกัน ทั้งในการเข้ารหัสและถอดรหัส — เรียกอีกอย่างว่า Secret Key Cryptographyวิทยาการเข้ารหัสลับ — การศึกษาเทคนิคสำหรับการสื่อสารอย่างปลอดภัยในที่ที่มีผู้ไม่หวังดี หรือ Private Key Cryptographyวิทยาการเข้ารหัสลับ — การศึกษาเทคนิคสำหรับการสื่อสารอย่างปลอดภัยในที่ที่มีผู้ไม่หวังดี:

Plaintext + Secret Key → [Encryption] → Ciphertext
Ciphertext + Secret Key → [Decryption] → Plaintext

ข้อดี:

  • เร็วและมีประสิทธิภาพสูง — เหมาะกับข้อมูลปริมาณมาก
  • ใช้ทรัพยากร CPU น้อย
  • Algorithm ที่ดี (เช่น AESAdvanced Encryption Standard — มาตรฐานการเข้ารหัสแบบสมมาตรที่ใช้กันแพร่หลายที่สุด มีขนาดคีย์ 128, 192 หรือ 256 บิต) ใช้ Hardware Acceleration ใน CPU สมัยใหม่ (AESAdvanced Encryption Standard — มาตรฐานการเข้ารหัสแบบสมมาตรที่ใช้กันแพร่หลายที่สุด มีขนาดคีย์ 128, 192 หรือ 256 บิต-NI)

ข้อเสีย:

  • ปัญหา Key Distribution — ผู้ส่งและผู้รับต้องมี Key ที่เหมือนกัน การส่ง Key อย่างปลอดภัยเป็นความท้าทาย
  • ไม่รองรับ Non-repudiation — Key เหมือนกันทั้งสองฝ่าย พิสูจน์ไม่ได้ว่าใครเป็นผู้สร้างข้อความ
  • Scalability ต่ำ — ในระบบที่มี N คน ต้องจัดการ Key = N(N-1)/2 คู่

3.1 หลักการทำงาน

ใช้ Key สองตัว (Key Pair) ที่สัมพันธ์กันทางคณิตศาสตร์ แต่ไม่สามารถหา Key หนึ่งจากอีก Key หนึ่งได้ในเวลาอันสมควร:

Keyการเข้าถึงหน้าที่
Public Key (กุญแจสาธารณะ)เปิดเผยต่อสาธารณะ — ทุกคนรู้ได้ใช้สำหรับ Encryption (เข้ารหัส) และ Verification (ตรวจสอบลายเซ็น)
Private Key (กุญแจส่วนตัว)เก็บเป็นความลับ — เฉพาะเจ้าของเท่านั้นใช้สำหรับ Decryption (ถอดรหัส) และ Signing (ลงนาม)

การเข้ารหัส:

Plaintext + Public Key → [Encryption] → Ciphertext
Ciphertext + Private Key → [Decryption] → Plaintext

การลงนาม:

Message + Private Key → Signature
Message + Signature + Public Key → Verify (True/False)

ข้อดี:

  • ไม่มีปัญหา Key Distribution — Public Key เผยแพร่ได้อิสระ
  • รองรับ Digital Signatureลายเซ็นดิจิทัล — กลไกยืนยันตัวตนผู้ส่งและความถูกต้องของข้อมูล โดยใช้การเข้ารหัสแบบไม่สมมาตร และ Non-repudiation — Private Key มีคนเดียวเท่านั้น
  • Scalability ดี — N คน ใช้ Key Pair แค่ N ชุด

ข้อเสีย:

  • ช้ามาก — ช้ากว่า Symmetric 100-1,000 เท่า (ไม่เหมาะกับข้อมูลปริมาณมาก)

RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่ เป็นอัลกอริทึม Asymmetric ที่ใช้กันแพร่หลายที่สุด — พัฒนาโดย Ron Rivest, Adi Shamir, และ Leonard Adleman ในปี 1977

หลักการทางคณิตศาสตร์:

RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่ อาศัยความยากของการแยกตัวประกอบของจำนวนเฉพาะขนาดใหญ่ (Integer Factorization Problem):

1. เลือกจำนวนเฉพาะขนาดใหญ่ 2 ตัว: p และ q
2. คำนวณ n = p × q                     (n = Modulus — ใช้ใน Public และ Private Key)
3. คำนวณ φ(n) = (p-1)(q-1)            (Euler's Totient)
4. เลือกค่า e: 1 < e < φ(n), gcd(e, φ(n)) = 1   (Public Exponent — มักใช้ 65537)
5. คำนวณ d ≡ e^(-1) mod φ(n)         (Private Exponent)

Public Key:  (n, e)
Private Key: (n, d)

Encryption:   c = m^e mod n           (m = plaintext, c = ciphertext)
Decryption:   m = c^d mod n

    ─────                                ───
    เลือก a (Private Key)                เลือก b (Private Key)
    คำนวณ A = g^a mod p                  คำนวณ B = g^b mod p
              │                                   │

4.5 HMACHash-based Message Authentication Code — กลไกตรวจสอบความถูกต้องของข้อความโดยใช้ Hash Function ร่วมกับ Secret Key (Hash-based Message Authentication Code)

HMACHash-based Message Authentication Code — กลไกตรวจสอบความถูกต้องของข้อความโดยใช้ Hash Function ร่วมกับ Secret Key ใช้ฟังก์ชัน Hash ร่วมกับ Secret Key เพื่อให้ทั้ง Authentication (ยืนยันตัวตนผู้ส่ง) และ Integrity (ตรวจสอบว่าข้อมูลไม่ถูกเปลี่ยนแปลง):

HMAC(K, m) = H((K' ⊕ opad) || H((K' ⊕ ipad) || m))

โดยที่:
H   = Hash Function (SHA-256)
K   = Secret Key
K'  = Key ที่ถูกปรับขนาดให้เท่ากับ Block Size
opad = 0x5c5c5c... (Outer Padding)
ipad = 0x363636... (Inner Padding)
m   = Message
||  = Concatenation
⊕   = XOR

5. Digital Signatureลายเซ็นดิจิทัล — กลไกยืนยันตัวตนผู้ส่งและความถูกต้องของข้อมูล โดยใช้การเข้ารหัสแบบไม่สมมาตร

5.1 หลักการทำงาน

Digital Signatureลายเซ็นดิจิทัล — กลไกยืนยันตัวตนผู้ส่งและความถูกต้องของข้อมูล โดยใช้การเข้ารหัสแบบไม่สมมาตร ใช้ Asymmetric Encryption แบบกลับด้าน — ลงนามด้วย Private Key ตรวจสอบด้วย Public Key:

การลงนาม (Signing):

1. Hash(Message) → Digest                         (ย่อขนาดข้อความ)
2. Digest + Private Key → Signature               (เข้ารหัส Digest ด้วย Private Key)

การตรวจสอบ (Verification):

1. Signature + Public Key → Digest1               (ถอดรหัส Signature)
2. Hash(Message) → Digest2                          (Hash ข้อความใหม่)
3. เปรียบเทียบ Digest1 == Digest2?                  (ถ้าเท่ากัน → Signature ถูกต้อง)

5.2 ทำไมต้อง Hash ก่อนเซ็น?

  • ประหยัดเวลา — การเซ็นข้อมูลขนาดใหญ่ด้วย Asymmetric ช้ามาก — Hash ทำให้ข้อมูลเล็กลง (256 บิต)
  • เพิ่มความปลอดภัย — Hash Function ทำลายโครงสร้างของข้อมูล — ป้องกันการโจมตีทางคณิตศาสตร์

5.5 Sign then Encrypt (แนวทางปฏิบัติ)

ในการส่งข้อมูลที่ต้องการทั้ง Authentication (เซ็น) และ Confidentiality (เข้ารหัส) ต้องทำ Sign แล้ว Encrypt:

ส่งข้อมูลที่ปลอดภัย:
1. ผู้ส่ง: Sign ข้อความด้วย Private Key
2. ผู้ส่ง: Encrypt (ข้อความ + Signature) ด้วย Public Key ของผู้รับ
3. ผู้รับ: Decrypt ด้วย Private Key ของตนเอง — ได้ข้อความ + Signature
4. ผู้รับ: Verify Signature ด้วย Public Key ของผู้ส่ง

ส่ง → Alice Sign → Alice Encrypt → Bob Decrypt → Bob Verify → Bob

เฉลย: Cryptographic Hash (SHASecure Hash Algorithm — กลุ่มฟังก์ชัน Hash มาตรฐานที่ออกแบบโดย NIST ใช้สร้างค่ากระจายเฉพาะของข้อมูล-256) ออกแบบมาให้ เร็ว — ซึ่งเป็นจุดอ่อนสำหรับ Password Hashing เพราะผู้โจมตีสามารถลอง Password ได้หลายพันล้านครั้งต่อวินาทีด้วย GPU/ASIC Password Hashing (bcrypt, argon2) ออกแบบให้ ช้า — ใช้ Work Factor ที่ปรับได้, ใช้ Memory มาก (Memory Hard) ทำให้การโจมตีด้วย GPU/ASIC ทำได้ยาก ความแตกต่างหลัก: (1) ความเร็ว — SHASecure Hash Algorithm — กลุ่มฟังก์ชัน Hash มาตรฐานที่ออกแบบโดย NIST ใช้สร้างค่ากระจายเฉพาะของข้อมูล-256 เร็วกว่ามาก; (2) Parallel — SHASecure Hash Algorithm — กลุ่มฟังก์ชัน Hash มาตรฐานที่ออกแบบโดย NIST ใช้สร้างค่ากระจายเฉพาะของข้อมูล-256 เหมาะกับ GPU/ASIC; (3) Salt — จำเป็นสำหรับ Password Hashing, ไม่จำเป็นสำหรับ SHASecure Hash Algorithm — กลุ่มฟังก์ชัน Hash มาตรฐานที่ออกแบบโดย NIST ใช้สร้างค่ากระจายเฉพาะของข้อมูล-256; (4) Adjustable Work Factor — มีเฉพาะ Password Hashing; (5) Argon2 เป็น Password Hashing Algorithm ที่แนะนำล่าสุด — ชนะ Password Hashing Competition (2015)

Post-Quantum Cryptographyการเข้ารหัสหลังยุคควอนตัม — อัลกอริทึมที่ต้านทานการโจมตีด้วยคอมพิวเตอร์ควอนตัม คืออะไร? อัลกอริทึมใดที่ถูกคัดเลือกโดย NIST และใช้แทน RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่/ECCElliptic Curve Cryptography — การเข้ารหัสแบบไม่สมมาตรที่ใช้โครงสร้างของElliptic Curve ให้ความปลอดภัยเทียบเท่า RSA แต่ใช้คีย์สั้นกว่า ได้?

เฉลย: Post-Quantum Cryptographyการเข้ารหัสหลังยุคควอนตัม — อัลกอริทึมที่ต้านทานการโจมตีด้วยคอมพิวเตอร์ควอนตัม (PQC) คืออัลกอริทึมที่ทนทานต่อการโจมตีด้วย Quantumควอนตัม — การใช้หลักการทางกลศาสตร์ควอนตัมในการประมวลผล ซึ่งสามารถทำลายระบบเข้ารหัสปัจจุบันได้ Computer — ปัจจุบัน RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่ และ ECCElliptic Curve Cryptography — การเข้ารหัสแบบไม่สมมาตรที่ใช้โครงสร้างของElliptic Curve ให้ความปลอดภัยเทียบเท่า RSA แต่ใช้คีย์สั้นกว่า ถูกทำลายด้วย Shor's Algorithm (Polynomial Time) อัลกอริทึมที่ NIST คัดเลือก (2024): CRYSTALS-Kyberอัลกอริทึม KEM มาตรฐาน Post-Quantum ของ NIST ที่ผ่านการคัดเลือกให้เป็นมาตรฐานในปี 2024 (FIPS 203) — KEMKey Encapsulation Mechanism — กลไกสำหรับส่งคีย์ลับระหว่างสองฝ่ายอย่างปลอดภัย ใช้ใน Post-Quantum Cryptography ใช้แทน RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่ Key Exchange และ ECDH CRYSTALS-Dilithiumอัลกอริทึมลายเซ็นดิจิทัลมาตรฐาน Post-Quantum ของ NIST ที่ผ่านการคัดเลือกให้เป็นมาตรฐานในปี 2024 (FIPS 204) — Digital Signatureลายเซ็นดิจิทัล — กลไกยืนยันตัวตนผู้ส่งและความถูกต้องของข้อมูล โดยใช้การเข้ารหัสแบบไม่สมมาตร ใช้แทน RSARivest-Shamir-Adleman — อัลกอริทึมเข้ารหัสแบบไม่สมมาตร อาศัยความยากของการแยกตัวประกอบจำนวนเฉพาะขนาดใหญ่ Sign, ECDSA, EdDSA FALCON (FIPS 205) — Signature สำหรับกรณีที่ต้องการ Size เล็ก SPHINCS+ (FIPS 205) — Stateless Hash-based Signature การเปลี่ยนผ่าน: ปัจจุบันใช้ Hybrid Mode (PQC + Traditional) เช่น X25519Kyber768 — NSA กำหนดให้ใช้ PQC ภายในปี 2030 (CNSA 2.0)
